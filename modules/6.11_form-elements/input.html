    Atrybuty

    accept
    Przyjmuje listę plików (MIME-type, rozszerzenie itd.), które można dodać przy type="file".
    autocomplete
    Nazwa pola dla przeglądarki, żeby wiedziała, co ma podpowiadać. Lista dostępnych nazw: HTML attribute: autocomplete.
    checked
    W przypadku type="radio" lub type="checkbox" pojawia się, jeśli pole jest zaznaczone.
    disabled
    Oznacza pole jako nieaktywne.
    list
    Powiązanie z id elementu datalist, pokazuje podpowiedzi z listy w w/w polu.
    min / max
    Odpowiednio minimalne lub maksymalne wartości, w zależności od typu pola.
    minlength / maxlength
    Odpowiednio minimalna/maksymalna długość (ilość znaków) wartości pola. Raczej unikamy ze względów dostępności.
    multiple
    W przypadku type="file | email" pozwala podać więcej niż jedną wartość.
    name
    Unikalna nazwa pola, konieczna przy wysyłaniu formularza na serwer.
    pattern
    Wyrażenie regularne do walidacji wartości pola.
    placeholder
    Wartość zastępcza, pokazywana kiedy input jest pusty.
    readonly
    Blokuje edycję wartości pola.
    required
    Wskazuje, czy pole jest obowiązkowe.
    step
    Stopień inkrementacji wartości.
    type
    Typ pola: https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input#input_types.
    value
    Wstępna1 wartość pola. Ważne zwłaszcza dla type="checkbox | radio".

    Przypisy:

    1. W czystym HTML wartość wstępna, natomiast później - w React itd - wartość bieżąca.
