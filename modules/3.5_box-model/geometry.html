    Właściwość padding 

    padding: <length> | <percentage>

    Odstęp między krawędzią a zawartością elementu. Uwaga: procentowa wartość wyliczana jest na podstawie liniowego wymiaru1 rodzica, nie elementu, który stylujesz.

    padding: 1em;
/* = */
padding: 1em 1em 1em 1em;
/* = */
padding-inline: 1em;
padding-block: 1em;
/* = */
padding-top: 1em;
padding-right: 1em;
padding-bottom: 1em;
padding-left: 1em;
        
    padding: 2em 0.5em;
/* = */
padding: 2em 0.5em 2em 0.5em;
/* = */
padding-inline: 0.5em;
padding-block: 2em;
/* = */
padding-top: 2em;
padding-right: 0.5em;
padding-bottom: 0.5em;
padding-left: 1em;
        
    padding: 2em 1em 3em;
/* = */
padding: 2em 1em 3em 1em;
/* = */
padding-inline: 1em;
padding-block: 2em 3em;
/* = */
padding-top: 2em;
padding-right: 1em;
padding-bottom: 3em;
padding-left: 1em;

    Właściwość margin

    margin: <length> | <percentage> | auto

    Składnia, shorthand i wyliczenia procentowe zachowują się tak samo, jak w przypadku paddingu.

    Załamywanie się marginesów, tzw. margin collapse

    Jeśli sąsiadujące ze sobą elementy w układzie blokowym mają marginesy, są one załamywane. Oznacza to, że algorytm je porównuje i wybiera jeden - większy - z nich. W poniższym przykładzie elementy nie będą miały 5em marginesu między sobą, tylko 3. Więcej w tym artykule na MDN.

    ul > li {
  margin-block: 2em 3em;
  /* = margin-top: 2em; margin-bottom: 3em; */
  /* = margin: 2em 0 3em; */
}

    "Wypadanie" marginesów

    Innym przypadkiem marginesów zachowujących się nieco nieintuicyjnie jest tzw. wypadanie marginesów (również związane z margin collapsing). Polega ono na tym, że element znajdujący się na "brzegu" elementu w układzie blokowym (czyli pierwszy lub ostatni element) będzie rozpychał się swoim marginesem poza rodzica.

    ul > li {
  margin-top: 5em;  /* = margin-block-start: 5em; */
}
